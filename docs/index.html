How It Works

From registration to symbol ID in seconds

OIS combines Unicode symbol generation with privacy-first profile management. Your symbol ID is yours forever — no usernames to squabble over, no handles to lose.

1

Register with email + TOTP

Create an account secured by two-factor authentication from day one. No phone number needed — just your authenticator app.

2

Generate your symbol ID

The engine samples three symbols from a 5,000-symbol Unicode pool, checks global uniqueness, and pairs each with a word alias.

⚙ ‑ 🌊 ‑ 🔥  ·  gear-wave-fire
3

Build your profile, control visibility

Add fields like display name, bio, and location. Each field has a per-field visibility setting — public, private, or future consent-based sharing.

4

Share your ID, get discovered

Anyone can search by symbol or alias through the public API. Your ID is stable and permanent — the foundation of a future federated identity layer.

Features

Built for privacy, built for scale

OIS ships a full stack — identity engine, REST API, and React UI — ready to self-host or extend.

🔤

Unicode Symbol IDs

Three symbols from a carefully curated, culturally neutral pool of 5,000+ characters. Memorable, copy-pasteable, renderable anywhere.

🔐

TOTP-first Security

Two-factor authentication is mandatory at registration, not an afterthought. JWT sessions with reset flows built in.

👁

Per-field Visibility

Each profile field carries its own visibility flag. Public data is searchable; private data never leaves your account.

🔍

Public Search API

Case-insensitive search across symbol IDs and aliases. Designed for embedding in other applications and future protocol clients.

⚡

FastAPI + React Stack

High-performance Python backend with async support. Vite-powered React frontend. Full Docker Compose setup for local development.

🌐

Open Protocol Vision

OIS is designed to evolve into a decentralized, federated identity layer — not just another SaaS auth product.

Roadmap

Where we are and where we're going

OIS is in active early development. The core identity engine and API are in place — bigger features are on the horizon.

Active MVP

Core Identity System

  • Unicode symbol + alias engine (5,000-symbol pool)
  • Registration, login, TOTP 2FA
  • Profile CRUD with per-field visibility
  • Public search API
  • React + Vite frontend
  • Docker Compose full-stack setup
Next Phase 2

Privacy & Access Control

  • Passkey / WebAuthn support
  • Consent-based data sharing between identities
  • Audit logs for profile access
  • Rate limiting and abuse protection
Later Phase 3

Decentralization & Federation

  • Distributed ID generation (no single point of failure)
  • Federation protocol — run your own OIS node
  • Cross-node identity resolution
  • Open specification for third-party implementations
